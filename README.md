# Plot2Eq 📈

**Plot2Eq** — это Machine Learning пет-проект, цель которого: распознавание математической формулы по нарисованному от руки графику функции $y = f(x)$.

В перспективе это Web-приложение, где пользователь рисует кривую на канвасе, а система выдает точную математическую формулу, описывающую этот график.

## 💡 Архитектура и идея проекта

Задача предсказания точной формулы (с конкретными константами, например $y = 2.5 \sin(3x) + 1.2$) "в лоб" очень тяжела для нейросетей. Поэтому проект элегантно разделен на две логические части:

1. **ML часть (Предсказание скелета):**
   Нейросеть (Point-to-Sequence архитектура: 1D-CNN Encoder + Transformer Decoder) "смотрит" на форму графика и предсказывает общий математический скелет с плейсхолдерами `C`. 
   *Пример выхода: `C * sin(C * x) + C`*
2. **Math часть (Подбор констант):**
   Классические методы оптимизации (например, `scipy.optimize.curve_fit`) берут предсказанный скелет и подбирают идеальные числовые значения констант под нарисованные точки.

## 📊 Датасет

Для обучения модели был сгенерирован и тщательно очищен от визуальных коллизий огромный датасет (~1.47 млн уникальных примеров).

## 🐳 Запуск генерации датасета в Docker

Если вы хотите сгенерировать сырые данные самостоятельно на сервере, в проекте предусмотрен Docker-образ для автономной генерации (символьной регрессии).

### 1. Сборка образа
Находясь в корне проекта (где лежит `Dockerfile`), выполните команду:
```bash
docker build -t plot2eq-gen .
```

### 2. Запуск генерации
⚠️ **Важно:** Обязательно используйте флаг `-v` для проброса папки. Иначе все сгенерированные чанки (`.pkl.gz`) навсегда удалятся при остановке контейнера!

```bash
docker run -d \
  --name data_generator \
  -v $(pwd)/data:/app/data \
  plot2eq-gen
```
* `-d` — запускает процесс в фоне (вы можете безопасно закрыть SSH/терминал).
* `-v $(pwd)/data:/app/data` — физически сохраняет данные в папку `data` на вашем хосте/сервере.

### 3. Мониторинг и управление

Посмотреть логи в реальном времени (увидеть прогресс-бар `tqdm` и скорость):
```bash
docker logs -f data_generator
```
*(Для выхода из логов нажмите `Ctrl+C` — генерация при этом **не** остановится).*

Посмотреть потребление CPU и оперативной памяти:
```bash
docker stats data_generator
```

Остановить генерацию:
```bash
docker stop data_generator
```

Запустить остановленную генерацию заново:
```bash
docker start data_generator
```

Удалить контейнер (если хотите пересобрать образ):
```bash
docker rm -f data_generator
```
